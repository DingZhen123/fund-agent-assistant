# RAG 测试问题样例

## 文档说明

本文收集用于验证资金系统 RAG 效果的测试问题。每个问题都可以用于检验知识检索、问题路由、答案引用和低置信度处理能力。

## 付款状态类问题

- EC2025 显示付款处理中是什么意思？
- 付款处理中是不是代表已经付款成功？
- 审批驳回后还能继续付款吗？
- 已付款是不是一定代表回单已经生成？
- 付款失败后系统会自动重试吗？
- 待付款和付款处理中有什么区别？

## 回单类问题

- 已付款但没有回单是什么原因？
- 回单一般什么时候生成？
- 为什么我能看到付款状态但不能下载回单？
- 回单下载提示文件不存在怎么办？
- 没有回单能不能说明没有付款？
- 回单可以重复下载吗？

## 异常处理类问题

- 付款失败后能不能直接重试？
- 付款处理中超过很久应该怎么处理？
- 银行返回成功但系统还是处理中怎么办？
- 多笔付款同时失败应该怎么排查？
- 收款账户信息错误应该怎么处理？
- 怎么避免付款处理中重复付款？

## 权限类问题

- 为什么我看不到某个付款单据？
- 刚开通权限为什么还不能下载回单？
- 谁可以执行付款操作？
- 审批人为什么看不到待审批任务？
- 普通业务人员可以重试付款吗？

## 组合类问题

- EC2025 已付款但为什么没有回单？
- 单据已付款，但用户下载回单提示没有权限，应该怎么排查？
- 付款处理中超时，同时银行没有返回结果，能不能重新付款？
- 付款失败原因是户名不匹配，是否可以直接重试？
- 用户刚被授权后仍看不到单据，可能是什么原因？

## 期望回答原则

回答应优先基于知识库内容，不应编造未检索到的系统配置。涉及具体单据状态时，应提示需要调用付款查询工具获取实时状态。涉及权限问题时，应说明可能原因和建议排查路径。
