# 付款回单生成与下载规则

## 文档说明

本文说明付款回单的生成条件、生成时机、下载权限、常见异常和处理流程。适用于回答“回单什么时候生成”“已付款为什么没有回单”“回单下载失败怎么办”等问题。

## 回单生成前置条件

付款回单生成通常需要同时满足以下条件：

- 付款单据状态为已付款。
- 银行或付款通道已经返回有效流水。
- 回单生成任务已经完成处理。
- 单据没有被冲正、撤销或标记为异常。
- 当前业务系统可以访问回单文件存储服务。

如果付款状态仍是待审批、审批中、待付款或付款处理中，系统通常不会生成正式银行回单。

## 回单生成时间

正常情况下，付款成功后系统会等待银行返回流水。银行流水返回后，回单生成任务会异步生成电子回单文件。

同银行付款通常较快生成回单，跨行付款可能存在延迟。部分银行在非工作时间、夜间批处理、节假日或通道维护期间返回回单会更慢。

如果付款成功后短时间内没有回单，通常不代表付款失败。建议先等待系统约定的回单同步周期，再判断是否异常。

## 已付款但没有回单的常见原因

已付款但没有回单，常见原因包括：

- 银行流水已经确认付款成功，但银行回单文件尚未返回。
- 回单生成任务排队或执行延迟。
- 回单服务异常，导致文件没有成功生成。
- 文件存储服务异常，导致回单生成成功但无法下载。
- 当前用户没有该单据或该组织下的回单下载权限。
- 单据处于特殊付款通道，回单需要人工补录或线下获取。

排查时应先确认付款流水状态，再确认回单生成任务状态，最后检查文件权限和下载链路。

## 回单下载权限

回单文件通常只对有权限的资金人员、财务人员或单据所属组织用户开放。用户能看到付款状态，不代表一定可以下载回单。

如果用户提示“没有权限下载回单”，应检查用户是否属于单据所属组织、是否具备回单查看权限、是否具备附件下载权限，以及权限变更是否已经同步到资金系统。

## 回单下载失败处理

回单下载失败时，应区分以下几类情况：

- 如果提示没有回单，说明系统尚未生成或未同步回单文件。
- 如果提示没有权限，说明用户权限不足或权限未同步。
- 如果提示文件不存在，可能是文件存储记录和实际文件不一致。
- 如果提示下载超时，可能是文件服务、网络或对象存储访问异常。

处理时建议先重新查询单据回单状态。如果回单状态为已生成但下载失败，应检查文件 URL、存储服务和下载权限。

## 处理流程

当用户反馈已付款但没有回单时，建议按以下流程处理：

1. 查询付款单据状态，确认是否为已付款。
2. 查询银行流水状态，确认是否已经返回有效流水。
3. 查询回单生成状态，确认是否已经生成文件。
4. 如果未生成，等待下一次回单同步任务或触发补偿任务。
5. 如果已生成但无法下载，检查文件存储和用户权限。
6. 如果银行未返回回单，联系银行或付款通道支持确认。

## 常见问答

### 为什么付款成功后回单不是马上生成？

因为回单生成依赖银行流水和回单文件返回，系统通常采用异步任务处理。付款成功和回单生成不是同一个动作。

### 回单可以重复下载吗？

可以。只要回单文件存在且用户有权限，通常可以重复下载。

### 回单文件名一般是什么格式？

常见格式为付款单据号加回单类型，例如 `EC2025_银行回单.pdf`。具体格式可能因银行和业务系统配置不同而不同。

### 没有回单能证明没有付款吗？

不能。是否付款成功应以付款状态、银行流水和银行返回结果为准。没有回单只能说明回单文件尚未生成、未同步或用户无权限查看。
